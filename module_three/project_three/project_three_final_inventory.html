Project Three: Final Inventory

The Bugg brothers seem to have gotten the hang of the backend for their inventory system. All that’s left is the frontend, which they can’t seem to get right. They were hoping to expand on your original frontend mock-up, but it wasn’t going well. They were wondering if you would be able to write the code for the final UI design since you did such a good job last time!

For this project, design the final UI for the Bugg brothers’ inventory system. This time, the table should be complete, as shown in the example below. The Bugg brothers are hoping to branch out after getting their grocery store set up, so they want their inventory system to be able to accommodate fields with larger widths. The system should ask for the name, unit price, unit weight, and quantity of each product. It should also ask whether they are actively selling each product as new products are always put into the system before they are shipped out. It only needs to ask about three products like last time since the Bugg brothers can handle putting the UI together with their backend. Since it’s the final iteration, the table should make sure to accept and display monetary values and quantities the way one would expect!

The Bugg brothers thank you for everything you’ve done for them!
