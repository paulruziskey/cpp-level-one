Project One: Card Guesser

Project Two ❯

In this project, you’ll write code to reproduce the game shown in the video. For clearing the console to work, the CLion terminal needs to be set to emulate regular terminals, but other than that, there’s nothing strange that needs to be done.

Some things to keep in mind:

  • Clearing the console can be done by outputting the escape sequence "\033c". There are other variations of this since this is simply the version written in octal, so you can use any of them, but this is the simplest.
  • You should store as much about the game as possible in global constants so as many things as possible can be changed. The things that aren’t expected to change are that there are only red and black cards, and there are only numbered and face cards.
  • Use algorithms from the <algorithm> and <numeric> headers as much as possible.
  • Each hand should be sorted, first by value, then by suit. None of this should be hard-coded!
  • The suits are displayed differently than they’re represented.
  • The points received for each correctly guessed quality should depend on how unlikely the quality the player guessed was. For example, guessing that there will be five cards with values greater than ace isn’t that unlikely since 48/52 cards have values greater than an ace! You’re welcome to decide exactly how the points scale, but the points should be dynamic. The second video shows examples of this.
